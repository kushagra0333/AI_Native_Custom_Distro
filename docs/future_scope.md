# Future Scope

## Purpose

This document describes the major capabilities planned after Version 1. These items are not required to declare the first release complete, but they shape how the platform should remain modular.

## Advanced Interaction Modes

Future versions can add:

- wake-word and speech-to-text interaction
- richer graphical dashboards
- editor-integrated task views
- voice-first workflows for hands-free usage

These features should build on the existing local API rather than bypassing the daemon.

## Larger Codebase Support

After v1, codebase understanding can expand toward:

- larger repositories
- multi-language project support
- stronger retrieval and indexing strategies
- better symbol-aware chunking
- more advanced validation of code edits

This expansion depends on keeping the current indexing pipeline modular.

## Plugin Ecosystem Expansion

The long-term plugin direction includes:

- Docker as a stable first extension
- databases and migrations
- cloud provider integrations
- CI/CD systems
- project templates and scaffolding packs

The plugin system is designed so these integrations do not require redesigning the daemon core.

## Stronger Agent Coordination

Once the core platform is stable, future versions may support:

- richer planning strategies
- better execution recovery
- more specialized analysis agents
- multi-stage workflows over longer task horizons

This should happen only after v1 proves that bounded orchestration is reliable.

## System Intelligence

Future releases may broaden the role of the platform inside the OS by adding:

- deeper performance diagnostics
- environment optimization suggestions
- developer workload profiling
- hardware-aware tuning

These features move the product closer to a full AI-native operating environment while still staying grounded in developer productivity.

## Community Growth

As an open-source platform, future growth should emphasize:

- contributor-friendly plugin interfaces
- stable API boundaries
- better documentation and examples
- extension points for new stacks and workflows

The goal is a platform that can evolve from a focused v1 into a broader ecosystem without losing its local-first and developer-centric identity.
